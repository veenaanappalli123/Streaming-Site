This project was developed as part of a second-year academic course. The main goal was to understand real-time web applications, clientâ€“server communication, and proper technical documentation.

## Contributions

The work included the following contributions:

- Backend development using FastAPI
- Implementation of WebSocket communication
- Frontend logic for video synchronization
- Integration of YouTube IFrame API
- Debugging network and synchronization issues
- Writing and organizing technical documentation

Each part of the system was tested individually before integrating the full workflow.

---

## Development Workflow

The development process followed these steps:

1. Initial project structure setup
2. Backend API development
3. Frontend UI and logic implementation
4. WebSocket synchronization
5. Debugging and issue resolution
6. Documentation writing
7. Final testing and validation

This structured approach helped identify and fix issues efficiently.

---

## Challenges Faced

Several challenges were encountered during development:

- WebSocket configuration issues
- Windows port conflicts
- Real-time synchronization logic errors
- YouTube embed and playback restrictions

Each issue was resolved through systematic debugging and research.

---

## Version Control

GitHub was used for version control throughout the project.

Best practices followed:
- Meaningful commit messages
- Committing only necessary files
- Avoiding environment or temporary files
- Keeping the repository clean and organized
